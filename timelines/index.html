    ⏱️ Study Timelines & Benchmarks

    Based on "You Suck At Coding Interviews" guide recommendations.

    DSA Study Timeline: 115 Hours Total

    Phase 1: Foundations (0-35 hours)

    Material: Grokking patterns course (skip DP for now)
    Goal: Master 80%+ of patterns

    Hours Check-in Milestone
    5h Getting started Completed: 2-3 pattern types
    10h Foundations Easy problems < 15 min
    20h Pattern lock-in Easy problems < 10 min
    30h Flow state Medium problems possible
    35h Checkpoint 60% of Grokking complete

    What You Can Do: - [ ] Identify pattern from problem name - [ ] Write multiple approaches - [ ] Trace through examples on paper - [ ] Explain preconditions

    Red Flags: - ❌ Still writing brute force after 10h - ❌ Can't explain why pattern works - ❌ Learning new pattern seeps into old

    Phase 2: DP Mastery (35-50 hours)

    Material: Grokking DP course
    Goal: Can solve DP problems independently

    Hours Check-in Milestone
    35h DP start Understand top-down approach
    40h Mid-phase Can build memoization table
    45h Near end Can solve 70% of medium DP
    50h Complete DP patterns mastered

    Why Separate: DP conceptually different, needs focus

    What You Can Do: - [ ] Identify overlapping subproblems - [ ] Build recurrence relation - [ ] Choose top-down vs bottom-up - [ ] Optimize space

    Red Flags: - ❌ Still looking at solutions after 40h - ❌ Code works but can't explain why - ❌ Memoization table approach fuzzy

    Phase 3: Advanced (50-85 hours)

    Material: EPI chapters 5-16 + LeetCode medium
    Goal: All problems solvable (speed may vary)

    Hours Check-in Milestone
    50h 2/3 done All patterns understood
    60h 70% there Most mediums solvable
    70h 80% there Hard problems attempted
    80h Near ready Mostly consistent
    85h Checkpoint Ready for practice phase

    What You Can Do: - [ ] Solve any medium problem given unlimited time - [ ] Identify pattern quickly - [ ] Only stuck on implementation details

    Red Flags: - ❌ Still forgetting patterns - ❌ Can't solve 50% of mediums - ❌ Solution time unpredictable (20-60 min variance)

    Phase 4: Practice & Interview Prep (85-115 hours)

    Material: LeetCode mediums/hards mixed
    Goal: Interview-ready consistency

    Hours Check-in Milestone
    85h Start Solve random 10 problems
    95h 10% done 70% success on first try
    100h 20% done All solvable in < 30 min
    105h 30% done Behavioral + technical ready
    110h 40% done Mock interviews clean
    115h Complete Interview ready

    What You Can Do: - [ ] Solve 70% of random problems first try - [ ] Speed: easy < 5 min, medium < 20 min - [ ] Explain solution clearly - [ ] Handle follow-up questions

    Red Flags: - ❌ Still making same mistakes - ❌ Solution time increasing - ❌ Confidence decreasing

    System Design Timeline: 30+ Hours

    Phase 1: Study Solutions (0-15 hours)

    Material: Grokking System Design interview course (5-10 case studies)
    Goal: Learn patterns

    Hours Case Studies Milestone
    3h 1 (URL Shortener) Understand HLD/LLD structure
    6h 2 (Twitter) See pattern: API → workers → DB
    9h 3 (YouTube) Understand caching strategy
    12h 4 (Uber real-time) Understand WebSocket patterns
    15h 5+ (Your choice) Pattern recognition locked in

    What You Learn: - HLD (High-level design) components - LLD (Low-level design) code patterns - Tradeoff thinking - Capacity calculations

    Red Flags: - ❌ Just copying Grokking solutions - ❌ Can't explain why choice was made - ❌ Can't calculate capacity yourself

    Phase 2: Understand Theory (15-25 hours)

    Material: DDIA (Designing Data-Intensive Applications) key chapters
    Goal: Understand WHY

    Hours Reading Deep Dive
    2h Ch 5: Replication Multi-master, consistency models
    2h Ch 6: Partitioning Sharding strategies, rebalancing
    2h Ch 9: Consistency & Consensus CAP theorem, algorithms
    2h Ch 11: Stream Processing Events, message queues
    2h Review + Synthesis Connect theory to case studies

    What You Learn: - WHY replication? (High availability, read scaling) - Consistency models (strong, eventual, causal) - Sharding strategies (hash, range, directory) - Failure modes and recovery

    Red Flags: - ❌ Too abstract, can't connect to systems - ❌ Only reading, not annotating

    Phase 3: Reverse Engineer (25-30 hours)

    Material: Design similar systems without reading solutions
    Goal: Can apply independently

    Hours Exercise Difficulty
    2h Design URL Shortener Medium
    2h Design Twitter Hard
    2h Design YouTube Hard
    1h Design Messaging Medium
    2h Design Rate Limiter Easy

    Your Approach: 1. Design in text/whiteboard (no peeking) 2. Write down: requirements, capacity, HLD, LLD 3. Compare to reference solutions 4. 50% match? You understand core 5. 80%+ match? You're ready for interviews

    Red Flags: - ❌ Can't complete without help - ❌ Designs completely different from reference - ❌ Missing capacity calculations

    Phase 4: Interview Ready (30h+)

    Material: Mock interviews, follow-up questions
    Goal: Handle real interviews

    Hours Activity Focus
    2-3h Full mock interview Time pressure, clarity
    1h Feedback review Identify weak areas
    1h Targeted study Fix weak areas
    Repeat Going forward 2-3 mocks/week

    What Success Looks Like: - [ ] Latency SLA defined within 1 min - [ ] Capacity calculated without mistakes - [ ] HLD clear, all components justified - [ ] Tradeoffs explicit - [ ] Failure scenarios identified - [ ] Can defend every choice

    Combined Study Schedule

    Month 1: DSA Foundations + System Design Intro

    Week 1-2: DSA Phase 1 (0-15h)
Week 3-4: DSA Phase 1 (15-35h) + System Design Phase 1 start (0h)

    Month 2: DSA DP + System Design Solutions

    Week 1: DSA Phase 2 (35-42h)
Week 2: DSA Phase 2 (42-50h)
Week 3-4: DSA Phase 3 start (50-60h) + System Design Phase 1 complete (15h)

    Month 3: DSA Advanced + System Design Theory

    Week 1-2: DSA Phase 3 (60-75h)
Week 3: DSA Phase 3 (75-85h) + System Design Phase 2 (20-25h)
Week 4: DSA Phase 4 start (85+) + System Design Phase 2 complete (25h)

    Month 4: DSA Practice + System Design Practice

    Ongoing: DSA Phase 4 (85-115h) + System Design Phase 3-4 (25-30h+)
2-3 mock DSA interviews/week
1-2 mock System Design interviews/week

    Benchmark Checkpoints

    At 20 Hours

    • [ ] DSA: Completed 20% of Grokking
    • [ ] Easy problems < 10-15 min
    • [ ] Can explain 3 patterns clearly
    • [ ] System Design: Not started (OK)

    At 50 Hours

    • [ ] DSA: Completed 50% of Grokking + EPI start
    • [ ] Medium problems 15-25 min
    • [ ] Can solve 80% first attempt
    • [ ] System Design: Completed 1-2 case studies

    At 100 Hours

    • [ ] DSA: 85+ hours complete, interview ready
    • [ ] Can solve 70% of 20 random problems first try
    • [ ] Speed consistent, few bugs
    • [ ] System Design: 2-3 case studies, theory started

    At 145 Hours (DSA 115 + System Design 30)

    • [ ] DSA: Interview ready ✅
    • [ ] System Design: Interview ready ✅
    • [ ] Mock interviews going well
    • [ ] Behavioral prep next

    Tips for Staying on Schedule

    1. Track daily: Log hours, problems solved, mistakes
    2. Adjust weekly: Behind? Add 5h next week
    3. Red flags: If behind at checkpoint, redo that phase
    4. Don't rush: Better to add weeks than skip depth
